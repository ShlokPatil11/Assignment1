# Employee Data Pipeline

A batch ETL pipeline that generates synthetic employee data, processes it with Apache Spark, and loads clean records into PostgreSQL — all orchestrated via Docker Compose.

---

## Architecture

```
generate_data.py  →  employees_raw.csv  →  Spark ETL Job  →  PostgreSQL (employees_clean)
```

---

| Tool | Version |

## Setup & Usage

### 1. Install Python dependencies

```bash
pip install -r scripts/requirements.txt
```

### 2. Generate raw employee data

```bash
python scripts/generate_data.py
```

This writes **1,050 records** (with intentional data quality issues) to `data/employees_raw.csv`.

### 3. Start the infrastructure

```bash
docker-compose up -d postgres spark-master spark-worker
```

Wait ~10 seconds for PostgreSQL to finish initializing (the schema is auto-applied via `sql/schema.sql`).

### 4. Run the Spark ETL job

```bash
docker-compose run spark-submit
```

This executes `spark/etl_job.py` which:
- Reads `data/employees_raw.csv`
- Cleans and transforms the data
- Writes the result to the `employees_clean` table in PostgreSQL

## Expected Output

You should see log lines similar to:

```
[Step 1 – Raw ingested] Row count: 1050
[Step 2a – After dedup on employee_id] Row count: ~950
[Step 2b – After dropping missing required fields] Row count: ~930
[Step 2c – After removing invalid emails] Row count: ~858
[Step 2d] Future hire_dates flagged and set to null: ~52
[Step 3 – Final clean dataset ready for load] Row count: ~858
ETL complete – data successfully written to employees_clean
```

To verify the data in PostgreSQL:

```bash
docker exec -it postgres psql -U postgres -d employeedb -c "SELECT count(*) FROM employees_clean;"
docker exec -it postgres psql -U postgres -d employeedb -c "SELECT salary_band, count(*) FROM employees_clean GROUP BY salary_band;"
```

## Stopping the Stack

```bash
docker-compose down          # Stop containers, keep volumes
docker-compose down -v       # Stop containers AND delete data volumes
```

## Troubleshooting

### ❌ Port already in use (e.g., 8080, 8081)

Another process is using the port. Either stop the conflicting process or change the host-side port in `docker-compose.yml`:

```yaml
ports:
  - "9090:8080"   # Change 9090 to any free port on your machine
```

### ❌ `bitnami/spark:3.4: not found`

The Bitnami Spark image was deprecated in late 2025. This project uses `apache/spark:3.4.1` instead. Ensure your `docker-compose.yml` references this image.

### ❌ Spark cannot connect to PostgreSQL

- Confirm the PostgreSQL container is healthy: `docker ps`
- Both containers must be on the same Docker network (`pipeline_network`).
- Verify credentials in `etl_job.py` match `docker-compose.yml` (`postgres/postgres/employeedb`).
- The JDBC URL must use the **container name** as hostname: `jdbc:postgresql://postgres:5432/employeedb`

### ❌ JDBC driver not found

The `--packages org.postgresql:postgresql:42.6.0` flag in `spark-submit`'s entrypoint downloads the driver at runtime. Ensure the container has internet access. If behind a proxy, configure `JAVA_OPTS` with proxy settings.

### ❌ `employees_clean` table does not exist

The schema is applied by mapping `sql/schema.sql` to `/docker-entrypoint-initdb.d/`. This only runs on a **fresh** volume. If you changed the schema after first run:

```bash
docker-compose down -v   # Wipe the volume
docker-compose up -d postgres
```

### ❌ Spark job exits immediately / `etl_job.py` not found

Verify the `./spark` directory is mounted correctly. The entrypoint expects the script at `/opt/spark/jobs/etl_job.py`.
